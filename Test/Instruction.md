
# Инструкция по начальной настройке программы

Следует выполнить эту инструкцию в строгом соответствии с указанным порядком.

## 1. Задание прав для работы с новым функционалом пользователям информационной базы.

Для начальной настройки работы необходимо пользователю с правами **Администратор** создать новых пользователей информационной базы и/или существующим пользователям дать права для работы с новым функционалом : **"Обслуживание клиентов"**, **"Расчет заработной платы"**. 

Для этого **Администратор** в панели разделов выбирает раздел ***"Администрирование"*** и в панели его функий команду ***"Настройки пользователей и прав"***. В открывшемся окне в группе команд ***"Пользователи"*** нажимает команду ***"Пользователи"***.

Далее в открывшемся окне **Администратор** по нажатию на кнопку ***"Создать"*** открывает окно для создания нового пользователя информационной базы, либо выбирает в списке существующего пользователя и по двойному щелчку мыши или нажатию кнопки ***Enter*** открывает окно для редактирования выбранного пользователя.

После задания имен , способа аутенцификации, контактной информации и записи этих данных по кнопке ***"Записать"***, **Администратор** по ссылке ***"Права доступа"*** открывает окно назначения прав доступа и задает для пользователя один или несколько профилей из списка ниже: **"Бухгалтер ИТ фирмы"**, **"Менеджер по заявкам на обслуживание"**, **"Кадровик-расчетчик"**, **"Специалист по обслуживанию заявок"**. Нажимает еще раз ***"Записать"*** и закрывает окно.

**"Бухгалтер ИТ фирмы"** работает с  документами ***"Реализация товаров и услуг"*** и выполняет ***"Массовое создании актов"*** за необходимый период времени.

**"Менеджер по заявкам на обслуживание"**  заводит клиентов и договора с ними, принимает от клиентов  новые заявки отражая их документом  ***"Обслуживание клиентов"***  и назначает на них специалистов.

**Специалист по обслуживанию заявок"** закрывает существующие заявки на ***"Обслуживание клиентов"*** (вносит сведения о выполненых работах, прикрепляет к документу скан акта о выполненых работах и проводит документ).

**"Кадровик-расчетчик"** формирует документы  ***"Начисление заработной платы"***, ***"Начисление премий***, ***"Выплаты зарплат сотрудникам"***. Планирует график отпусков через документ ***"Графики отпусков"***. Заводит и редактирует данные по сотрудникам, и устанавливает условия оплаты труда сотрудников через документ ***"Установка окладов сотрудников"***.

## 2. Настройка телеграмм-бота для оповещения специалистов о появлении новых заявок или изменении существующих.

Для получения уведомлений сотрудниками с профилем ***Специалист по обслуживанию заявок*** о создании новых заявок на обслуживание клиентов или изменении уже существующих в группу  мессенджера *Телеграмм* **Администратору** необходимо выполнить настройки для работы с телеграмм-ботом: указать токен бота и идентификатор группы телеграмм для вышеописанных сотрудников. 

Для этого **Администратор** в панели разделов выбирает раздел ***"Администрирование"*** и в панели его функий команду ***"Дополнительные настройки подсистем"***. В открывшемся окне в группе команд ***"Телеграмм"*** задать в соответствующих полях токен телеграмм бота и идентификатор группы.

Так же **Администратор** может изменить настройки в двух других группах: ***Расчет ЗП"*** и ***"Обслуживание клиентов"*** при необходимости. Но следует придерживаться правила, что настройки в этих группах задают пользователи с правами ***"Кадровик-расчетчик"*** и ***"Бухгалтер ИТ фирмы"*** соответственно.

## 3. Настройка раздела "Расчет зароботной платы" 

Данная настройка производиться пользователем с правами ***"Кадровик-расчетчик"***.

Для этого указанный пользователь в панели разделов выбирает раздел ***"Расчет зароботной платы"*** и в панели его функий в группе команд ***Сервис*** нажимает на ***Дополнительные настройки подсистем***. В открывшемся окне в группе ***Расчеты ЗП*** в поле стафка НДФЛ указывает ставку в процентах для удержания НДФЛ по производимым начислениям. Далее ***Записать и закрыть***.

Далее в группе команд ***Справочники*** нажимает ***Сотрудники*** и заводит сотрудников. Для сотрудника указывается Фамилия Имя Отчество и категория сотрудника: *Управленец* или *Специалист*.

Далее в группе команд ***Документы*** нажимает ***Список документов установки окладов*** и в открывшемся окне создает документы ***"Установка окладов сотрудников"*** и проводит их. Для категорий сотрудников ***Управленец*** указывается оклад, а для категории "Специалист": оклад и процент от выполненых работ. Устанавливать условия оплаты сотрудникам можно один раз в месяц.

Далее в группе команд ***Справочники*** нажимает ***Графики работы*** и вводит данные графика работы организации.
Затем  в группе команд ***Сервис*** нажимает ***Заполнение графика*** и в открывшемся окне вводит данные графика работы организации за указанный период времени : 
* В поле ***Выбор периода*** указывается период времени для установки графика.
* В поле ***График работы*** указывается график работы организации из справочника ***Графики работы***.
* В поле ***Выходные дни*** указываются номер дня недели выходного дня или номера выходных дней недели через запятую для указанного в поле ***График работы*** графика. Например для Субботы указываем 6, а если выходными являются Суббота и Воскресенье, то указываем 6,7.

Далее нажимает ***Заполнить***.

## 4. Настройка раздела "Обслуживание клиентов"

Данная настройка производиться пользователем с правами ***Бухгалтер ИТ фирмы"***.

Для этого указанный пользователь в панели разделов выбирает раздел ***"Обслуживание клиентов"*** и в панели его функий в группе команд ***Сервис*** нажимает на ***Дополнительные настройки подсистем***. В открывшемся окне в группе ***Обслуживание клиентов*** в полях:
* ***Номенклатура абонентская плата*** выбирает номенклатуру  для подстановки взимания платы по договорам абоненского обслуживания клиентов в документах ***"Реализация товаров и услуг"***. Если такой номенклатуры нет, то создает ее и выбирает.
*  ***Номенклатура работы специалиста*** выбирает номенклатуру  для подстановки взимания платы за оплату времени выполненых работ  специалистом по заявкам клиента на обслуживание в документах ***"Реализация товаров и услуг"***. Если такой номенклатуры нет, то создает ее и выбирает. 

Далее ***Записать и закрыть***.

------

После выполнения всех вышеуказанных пунктов в строгом соответствии их порядку можно приступать к полноценному использованию нового функционала, созданного по техзаданию заказчика.
